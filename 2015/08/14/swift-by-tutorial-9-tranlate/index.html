Chapter 9 Swift vs. Objective-C Translate

这是Swift By Tutorials 的最后一张，我希望你有一个享受的旅途！

现在，你应该对Swift编程语言的知识有了一个很好的理解并且迫不及待的把它用去实践。我相信你会认同马特和自己,Swift是一个强大的和优雅的语言，它会取代不足和怪异的Objective-C语言。

在这最后一章，你将建立一个应用，涉及到许多目前为止你学习过得Swift主题。你将重现流行的棋盘游戏黑白棋,你的使用者可以使用他们的技能挑战电脑。

这个翻转棋应用已经用Objective-C语言实现并且发布在http://www.raywenderlich.com上作为一个系列:
• http://www.raywenderlich.com/29228/how-to-develop-an-ipad-board- game-app-part-12
• http://www.raywenderlich.com/29248/how-to-develop-an-ipad-board- game-app-part-22

等同Swift实现在本章将遵循完全和Objective-C相同的构建和运行步骤,可以让你比较两者的实现。

开始

这个游戏有很多资源包括启动项目的一部分。打开这个项目,打开Xcode,然后构建和运行。会呈现下面的屏幕给你:

花少许时间去熟悉下这个工程的结构。这个工程有个单一视图控制器的视图在Main.storyboard上并且有相关联的代码在ViewController.swift中。这个视图控制器有几个插座变量稍后你将使用在本教程中。

棋盘建模

在翻转棋中，玩家们的白边或者黑边的计分盘单元上面有个一个8×8网格板块。这个游戏的规则很简单，但是如果你不熟悉，在维基百科上面有很好的综述:
http://en.wikipedia.org/wiki/Reversi.

你的第一个任务是创建一个模型，代表棋盘上当前的状态，64个单元上每一个可能是有白棋子，黑棋子或没有棋子组成。这听起来好像Swift的枚举是个不错的选择。

添加一个Swift文件到Model组中并且命名为BoardCellState.swift，用下面的代码替换文件中得内容：

import Foundation enum BoardCellState 
{
    case Empty, Black, White 
}

上面定义了一个枚举和三个必要的状态为棋盘的每个单元。

你将在后续步骤中添加许多处理逻辑，从棋盘的方块中引导下一步，你将代表每一个方块的位置用一个匹配的数字:行和列，逻辑化这个简单类型的组合。

添加一个Swift文件到Model组中并且命名为BoardLocation.swift，添加下面的代码到文件中:

import Foundation struct BoardLocation 
{
    let row: Int, column: Int
    init(row: Int, column: Int) 
    {
        self.row = row
        self.column = column
    } 
}

上面代码定义了一个简单的结构体，有一个integer row 和 column组成的简单类型并且命名为 BoardLocation，你可能考虑过使用元组为达这个目的，例如: let position:(Int, Int),
但是你会发现结构体会更好描述，而且，结构体能够实现协议和方法，你将在后面使用这些特征在本章。

现在是时候添加棋盘了，添加一个Swift文件到Model组中，命名为：Board.swift，添加下面的代码到文件中:

import Foundation
class Board 
{
    private var cells: [BoardCellState] let boardSize = 8
    init ()
    {
        cells = Array(count: boardSize * boardSize,
        repeatedValue: BoardCellState.Empty)
    }
    subscript(location: BoardLocation) -> BoardCellState 
    { 
        get 
        {
            return cells[location.row * boardSize + location.column] 
        }
        set 
        {
            cells[location.row * boardSize + location.column] = newValue
        }
     }
}

定义了一个Board类，包含了一个单元数组和常整形指示棋盘的大小，初始化创建Cells数组，指定每一个元素的值为:BoardCellState.Empty

下标提供一个简单的方法获取或设置每一个cell的值，下面代码举例说明:
var board = Board()
board[BoardLocation(row: 4, column: 5)] = BoardCellState.White

然而，一个更加简明的格式是可能的，添加下面subscript到Board类:

subscript(row: Int, column: Int) -> BoardCellState 
{ 
    get 
    {
        return self[BoardLocation(row: row, column: column)] 
    }
    set 
    {
        self[BoardLocation(row: row, column: column)] = newValue
    }
 }

上面公开了一个下标，你可以使用如下:
var board = Board()
board[4, 5] = BoardCellState.White

下标允许你去创建一个非常灵活的API给你的Swift类，你也可以定义多个下标，允许你有多种方式去访问类的数据

为了演练你刚刚添加的代码，你需要创建UI来充分利用这个模型，然而，有一更直接并且立刻使用的机制-单元测试！

打开SwiftReversiTests.swift并且添加下面的方法到SwiftReversiTests类:

func test_subscript_setWithValidCoords_cellStateIsChanged() 
{
    let board = Board()
    // set the state of one of the cells
    board[4, 5] = BoardCellState.White
    // verify
    let retrievedState = board[4, 5]; 
    XCTAssertEqual(BoardCellState.White, retrievedState,"The cell should have been white!");
}

这个代码是一个简单的单元测试，演示下标的实现通过获取和设置Cells中某个的值，这实际测试了两个下标的实现，因为一个代表了另外一个。

在你编译工程之前，确保Swift文件你希望测试已经正确关联，你可以查看这些信息通过File Inspector，确保 BoardCellState.swift, BoardLocation.swift and Board.swift中每一个都选中SwiftReversi and SwiftReversiTests targets，否则测试不能够编译。
